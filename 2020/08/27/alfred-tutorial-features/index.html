Alfred使用不完全教程——Features

2020-08-27 杂项 Alfred, 教程 0 评论 字数统计: 770(字) 阅读时长: 2(分)

0. 引言

Alfred 是什么？用了 Alfred 有什么好处？
对我而言，触控板在进行操作时，大部分时间都用于“离开键盘+移动至触控板+目光搜索+调整点击”，重复的次数越多，越觉得浪费的时间越多。因此，Alfred 这种能够将操作映射或转换至键盘的工具便是效率提升的最佳选择。
类似的问题就不再赘述，本篇只提供一个我入手并学习使用 Alfred 的过程并记录下来，作为一个教程分享给大家。

1. 关于 Features

Features 在我看来是 Alfred 为免费版或者说小白用户提供的使用频次最高的功能集合，同时包括了一些软件的基础设置。注意，部分 feature 需要激活 Powerpack，接下来依次介绍一下。

1.1 Default Results

Default Results 主要设置的是 Alfred 在默认状况下的搜索结果，以及搜索源。

1.1.1 Applications

打开 Applications，选项为 Apps Matching 和 Match Application’s keyworkds in default results。

1. Apps Matching

Apps Matching 控制了搜索应用时的匹配规则，包含 Fuzzy capital letters 和 Full fuzzy match from word boundary，建议设置为默认的 Fuzzy capital letters。
Fuzzy capital letters 指只会匹配应用完整名字的首字母，以减少干扰。
Full fuzzy match from word boundary 指匹配应用名称内任意顺序的字母组合。
以 Visual Studio Code 为例，当设置为 Fuzzy capital letters 时，只有输入 vsc 的过程中可以检索到该应用，而 vsco 则不行。


反之，设置为 Full fuzzy match from word boundary 时，输入类似 vis/vst/vsoc 等 **”Visual Studio Code” **的顺序子串时，都可以搜索到该程序。干扰过大，因此建议选择默认。

2. Match Applicaiton’s keywords in default results

由于某些应用程序会有相关的元信息存在，开启此选项会对一应用程序的元信息同时进行搜索。

1.1.2 Essentials/Extras/Unintelligent

这两行选项主要设置了搜索的目标。Essentials 中，Preferences 指系统设置，Contacts 指通讯录。
Extras 则是指搜索结果显示的文件类型。如建议所言，建议只开启 Essentials 中的设置，如果需要搜索文件/文件夹，可以使用 open 关键词进行搜索。此外，调出 Alfred 后，单击一次空格键（或者输入’）也可以进入文件搜索模式。

Unintelligent 选项会搜索所有类型的文件，开启后会大大拖慢默认搜索的速度，建议使用 open 或者 find 命令进行文件相关的搜索。

1.1.3 Search Scope

Search Scope 定义了搜索的目标文件夹范围，可根据自己的需求添加或者删除。

1.1.4 Fallbacks


Fallbacks 设定为 Only 时，只有搜索不到目标时才会列出默认的搜索项。设置为 Intelligently 时，默认的搜索相会始终出现在搜索结果末尾。

Setup fallback results 设定了默认的搜索项。当前版本中，可添加的默认搜索项都为 Web Search。关于 Web Search，会在后面介绍。
